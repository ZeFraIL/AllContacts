# 📱 Документация Android-приложения AllContacts

________________________________________
🧾 Общая информация
**Название проекта:** AllContacts  
**Автор:** Zeev Fraiman  
**Дата:** 22.05.2024  
**Язык:** Java  
**Среда разработки:** Android Studio  
**Версия Android:** minSdk 16 / targetSdk 28  

________________________________________
🎯 Цель проекта
•	Приложение позволяет пользователю просматривать список всех контактов на устройстве.
•	Обеспечивает быстрый набор номера прямо из списка.
•	Отслеживает состояние входящих вызовов.
•	**Целевая аудитория:** Пользователи, которым нужен простой и быстрый доступ к контактам и функциям вызова.

________________________________________
📌 Требования к приложению
**Функциональные требования**
•	Запрос разрешений на чтение контактов и совершение звонков.
•	Извлечение имен и номеров телефонов из системной базы данных контактов.
•	Отображение контактов в удобном списке (ListView).
•	Совершение звонка при клике на контакт или ручном вводе номера.
•	Уведомление при входящем звонке.

**Нефункциональные требования**
•	**Производительность:** Быстрая загрузка списка контактов.
•	**Удобство использования:** Минималистичный интерфейс, понятный с первого взгляда.
•	**Надёжность:** Корректная обработка отсутствия разрешений.

________________________________________
🧠 Общая архитектура
•	**Выбранный подход:** Стандартная архитектура на основе компонентов Android (Activities, BroadcastReceivers).
•	**Почему выбран именно он:** Проект имеет небольшой масштаб, что делает использование сложных паттернов (вроде MVVM) избыточным на данном этапе.
•	**Основные компоненты системы:**
    – `MainActivity`: Управление UI и взаимодействием.
    – `PhonecallReceiver`: Обработка системных событий телефонии.
    – `ContactsContract`: Источник данных (системный провайдер).

🧩 Подробное описание классов
📌 Класс: MainActivity
**Роль:** Основной экран приложения.
**Ответственность:** Отображение списка контактов, обработка нажатий, запрос разрешений.
**Основные методы:**
- `onCreate()` — инициализация UI и запрос разрешений.
- `getContacts()` — чтение контактов через ContentResolver.
- `viewAllContacts()` — обновление адаптера списка.
- `callnow()` — запуск Intent для совершения звонка.

📌 Класс: PhonecallReceiver
**Роль:** Приемник широковещательных сообщений.
**Почему используется:** Для отслеживания состояния телефона (звонок, ожидание) в фоновом режиме.

________________________________________
🔄 Диаграмма работы приложения
1. Запуск приложения.
2. Запрос разрешений (CALL_PHONE, READ_CONTACTS).
3. Нажатие кнопки "Все контакты".
4. Извлечение данных из системы и отображение в списке.
5. Выбор контакта -> Автоматический набор номера.

🎨 UI/UX анализ
•	Интерфейс выполнен в стиле Material Design с использованием стандартных компонентов.
•	**Принципы:** Простота (один экран), Логичность (кнопка действия рядом с вводом), Доступность (крупный текст в списке).
•	**Что можно улучшить:** Добавить поиск по контактам, поддержку темной темы, использование RecyclerView вместо ListView.

________________________________________
⚙️ Работа с потоками
•	**Используются:** В данной версии операции выполняются в основном потоке (Main Thread).
•	**Оптимизация:** Для предотвращения ANR при очень большом количестве контактов рекомендуется перенести `getContacts()` в фоновый поток (Thread или AsyncTask/Coroutines).

________________________________________

•	**Способ доступа:** ContentResolver.
