# 📱 توثيق تطبيق أندرويد: AllContacts

________________________________________
🧾 معلومات عامة
**اسم المشروع:** AllContacts  
**المؤلف:** Zeev Fraiman  
**التاريخ:** 22 مايو 2024  
**اللغة:** Java  
**بيئة التطوير:** Android Studio  
**إصدار أندرويد:** minSdk 16 / targetSdk 28  

________________________________________
🎯 هدف المشروع
• يتيح للمستخدمين عرض القائمة الكاملة لجهات الاتصال المخزنة على الجهاز.
• يوفر وظيفة الاتصال السريع مباشرة من قائمة جهات الاتصال.
• يراقب حالات المكالمات الواردة ويقدم إشعارات.
• **الجمهور المستهدف:** المستخدمون الذين يبحثون عن أداة بسيطة وخفيفة لإدارة جهات الاتصال وإجراء المكالمات.

________________________________________
📌 متطلبات التطبيق
**المتطلبات الوظيفية**
• طلب أذونات التشغيل لقراءة جهات الاتصال وإجراء المكالمات.
• الاستعلام عن قاعدة بيانات جهات اتصال النظام للحصول على الأسماء وأرقام الهواتف.
• عرض جهات الاتصال في قائمة سهلة القراءة (ListView).
• بدء المكالمات الهاتفية عبر Intent عند اختيار جهة اتصال أو إدخال رقم.
• إظهار الإشعارات أثناء المكالمات الواردة.

**المتطلبات غير الوظيفية**
• **الأداء:** تحميل سريع لقائمة جهات الاتصال.
• **سهولة الاستخدام:** واجهة بسيطة وبديهية مكونة من شاشة واحدة.
• **الموثوقية:** التعامل القوي مع رفض الأذونات وقوائم جهات الاتصال الفارغة.

________________________________________
🧠 البنية العامة
• **النهج المختار:** بنية تعتمد على مكونات أندرويد القياسية (Activity، BroadcastReceiver).
• **سبب الاختيار:** المشروع بسيط ومباشر؛ الأنماط الأكثر تعقيداً مثل MVVM ستضيف تعقيداً غير ضروري لهذا الحجم.
• **مكونات النظام الأساسية:**
    – `MainActivity`: يدير واجهة المستخدم والمنطق الرئيسي.
    – `PhonecallReceiver`: يتعامل مع بث النظام المتعلق بالاتصالات الهاتفية.
    – `ContactsContract`: يعمل كمصدر للبيانات عبر ContentResolver.

________________________________________
🧩 وصف تفصيلي للفئات
📌 الفئة: MainActivity
**الدور:** واجهة المستخدم الأساسية.
**المسؤولية:** التعامل مع تفاعلات المستخدم، طلب الأذونات، وجلب/عرض جهات الاتصال.
**الطرق الرئيسية:**
- `onCreate()` — تهيئة الواجهات والتحقق من الأذونات.
- `getContacts()` — الاستعلام عن موفر جهات اتصال النظام.
- `viewAllContacts()` — تعبئة قائمة واجهة المستخدم بالبيانات المجلوبة.
- `callnow()` — تشغيل intent من نوع ACTION_CALL.

📌 الفئة: PhonecallReceiver
**الدور:** BroadcastReceiver لأحداث الهاتف.
**سبب الاستخدام:** لمراقبة حالة مكالمات الجهاز (رنين، خمول) والاستجابة وفقاً لذلك (مثل إظهار إشعار).

________________________________________
🔄 سير عمل التطبيق
1. تشغيل التطبيق.
2. طلب الأذونات (CALL_PHONE, READ_CONTACTS).
3. ينقر المستخدم على "كل جهات الاتصال" (All Contacts).
4. يجلب التطبيق البيانات من النظام ويعرضها في القائمة.
5. يختار المستخدم جهة اتصال -> يبدأ التطبيق مكالمة هاتفية.

🎨 تحليل UI/UX
• تستخدم الواجهة مكونات أندرويد القياسية لضمان الألفة.
• **المبادئ:** البساطة (شاشة واحدة)، المنطق (أزرار الإجراءات موضوعة بالقرب من حقول الإدخال)، إمكانية الوصول (عناصر قائمة كبيرة).
• **التحسينات:** إضافة وظيفة البحث، استخدام RecyclerView لأداء أفضل، وإضافة الوضع الداكن (Dark Mode).

________________________________________
⚙️ التعامل مع الخيوط (Threading)
• **الاستخدام الحالي:** الخيط الرئيسي (Main Thread) لواجهة المستخدم وجلب البيانات.
• **الوقاية:** لقواعد بيانات جهات الاتصال الكبيرة، يوصى بنقل `getContacts()` إلى خيط خلفي لمنع أخطاء ANR (التطبيق لا يستجيب).

________________________________________

• **التخزين:** يستخدم قاعدة بيانات جهات اتصال نظام أندرويد.
