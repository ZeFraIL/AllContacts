# 📱 תיעוד אפליקציית אנדרואיד: AllContacts

🧾 מידע כללי
**שם הפרויקט:** AllContacts  
**מחבר:** Zeev Fraiman  
**תאריך:** 22 במאי, 2024  
**שפה:** Java  
**סביבת פיתוח:** Android Studio  
**גרסת אנדרואיד:** minSdk 16 / targetSdk 28  

________________________________________
🎯 מטרת הפרויקט
• מאפשר למשתמשים לצפות ברשימה המלאה של אנשי הקשר השמורים במכשיר.
• מספק פונקציונליות חיוג מהיר ישירות מרשימת אנשי הקשר.
• מנטר את מצב שיחות נכנסות ומציג התראות.
• **קהל יעד:** משתמשים המחפשים כלי פשוט וקל לניהול וחיוג לאנשי קשר.

________________________________________
📌 דרישות האפליקציה
**דרישות פונקציונליות**
• בקשת הרשאות בזמן ריצה לקריאת אנשי קשר וביצוע שיחות.
• שליפת שמות ומספרי טלפון ממסד הנתונים של אנשי הקשר במערכת.
• הצגת אנשי קשר ברשימה נוחה לקריאה (ListView).
• ביצוע שיחות טלפון באמצעות Intent בעת בחירת איש קשר או הזנת מספר.
• הצגת התראות במהלך שיחות נכנסות.

**דרישות לא פונקציונליות**
• **ביצועים:** טעינה מהירה של רשימת אנשי הקשר.
• **חוויית משתמש:** ממשק פשוט ואינטואיטיבי במסך אחד.
• **אמינות:** טיפול נכון בסירוב להרשאות וברשימות אנשי קשר ריקות.

________________________________________
🧠 ארכיטקטורה כללית
• **גישה נבחרת:** ארכיטקטורה מבוססת רכיבי אנדרואיד סטנדרטיים (Activity, BroadcastReceiver).
• **סיבה לבחירה:** הפרויקט פשוט וממוקד; תבניות מורכבות יותר כמו MVVM יהיו מיותרות עבור קנה מידה זה.
• **רכיבי מערכת עיקריים:**
    – `MainActivity`: מנהל את ממשק המשתמש והלוגיקה העיקרית.
    – `PhonecallReceiver`: מטפל בשידורי מערכת הקשורים לטלפוניה.
    – `ContactsContract`: משמש כמקור הנתונים דרך ContentResolver.

🧩 תיאור מפורט של המחלקות
📌 מחלקה: MainActivity
**תפקיד:** ממשק המשתמש הראשי.
**אחריות:** טיפול באינטראקציות משתמש, בקשת הרשאות, ושליפה/הצגה של אנשי קשר.
**שיטות עיקריות:**
- `onCreate()` — מאתחל תצוגות ובודק הרשאות.
- `getContacts()` — שואל את ספק אנשי הקשר של המערכת.
- `viewAllContacts()` — מעדכן את רשימת התצוגה בנתונים שנשלפו.
- `callnow()` — מפעיל Intent מסוג ACTION_CALL.

📌 מחלקה: PhonecallReceiver
**תפקיד:** BroadcastReceiver לאירועי טלפוניה.
**סיבה לשימוש:** לניטור מצב השיחה של המכשיר (מצלצל, פנוי) ותגובה בהתאם (למשל, הצגת התראה).

________________________________________
🔄 זרימת עבודה של האפליקציה
1. הפעלת האפליקציה.
2. בקשת הרשאות (CALL_PHONE, READ_CONTACTS).
3. המשתמש לוחץ על "כל אנשי הקשר" (All Contacts).
4. האפליקציה שולפת נתונים מהמערכת ומציגה אותם ברשימה.
5. המשתמש בוחר איש קשר -> האפליקציה מבצעת שיחת טלפון.

🎨 ניתוח UI/UX
• הממשק משתמש ברכיבי אנדרואיד סטנדרטיים ליצירת מוכרות.
• **עקרונות:** פשטות (מסך יחיד), לוגיקה (כפתורי פעולה ליד שדות הקלט), נגישות (פריטי רשימה גדולים).
• **שיפורים:** הוספת פונקציונליות חיפוש, שימוש ב-RecyclerView לביצועים טובים יותר, והוספת מצב כהה (Dark Mode).

________________________________________
⚙️ עבודה עם תהליכים (Threads)
• **שימוש נוכחי:** Main Thread עבור ממשק המשתמש ושליפת נתונים.
• **מניעה:** עבור מסדי נתונים גדולים של אנשי קשר, מומלץ להעביר את `getContacts()` לתהליך רקע כדי למנוע שגיאות ANR.
